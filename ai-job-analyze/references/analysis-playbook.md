# 分析方法

## 报告目标

报告必须优先回答用户输入的分析问题，而不是泛泛总结岗位市场。每条核心洞察都应能追溯到岗位样本或失败来源。

## 建议结构

1. 一句话结论。
2. 样本范围和采集限制。
3. 公司差异。
4. 能力地图。
5. 城市、部门、岗位层级分布。
6. 面向用户问题的行动建议。
7. 附录：来源、失败项、样本限制。

## 组件边界

- 抓取组件只提供原始岗位和失败来源，不直接给结论。
- 字段归一组件只做结构化、去重、分类和证据摘录，不写报告叙事。
- 洞察组件负责回答用户问题，并为每条判断附上岗位样本或统计证据。
- 报告渲染组件只把结构化洞察转成 HTML 或 PNG，不重新解释数据。

HTML 是默认报告格式。PNG 是 HTML 的视觉导出版，不能成为唯一产物。

## 人工智能岗位识别

分三层识别：

- 明确人工智能：大模型、人工智能生成内容、智能体、机器学习、算法、推荐、计算机视觉、自然语言处理。
- 人工智能相关：智能化、搜索推荐、数据产品、内容理解、自动化。
- 噪声剔除：只在部门名或业务线中出现人工智能，但职责和要求无关的岗位应降权或排除。

## 证据要求

核心洞察至少关联以下证据之一：

- 岗位标题。
- 岗位来源 URL。
- 职责或要求中的关键词。
- 公司和渠道的统计差异。
